1.- Descripción de la tarea

Caso práctico

A María le han ofrecido un trabajo  para el mantenimiento de los equipos informáticos así como de la seguridad informática.El primer paso será adquirir los conocimientos básicos sobre seguridad y adoptar las pautas de seguridad básicas para poner el sistema en funcionamiento. Tras leer varios libros y preguntarle a su compañero Juan, descubre que la clave de todo sistema seguro es asegurar la fiabilidad, la confidencialidad, integridad y la disponibilidad de todos los elementos de un sistema.

María ya sabe que debe cumplir un sistema seguro, pero ahora tendrá que aprender que es lo que tiene que proteger, o cuáles son los puntos débiles de su sistema. 

¿Qué te pedimos que hagas?

  • Apartado 1: Pentesting, análisis y explotación de vulnerabilidades
    • Instala en una de las máquinas del entorno virtual proporcionado por el profesor, una herramienta para el análisis de vulnerabilidades (Nessus, OpenVas, Foca...)
    • Utiliza la herramienta elegida para realizar un análisis de las vulnerabilidades presentes en los equipos de la red del entorno virtual. Realiza una breve descripción de las vulnerabilidades presentes atendiendo a su nivel de criticidad.
    • Selecciona una de las vulnerabilidades detectadas (diferente al ejemplo mostrado en clase) y realiza un estudio de la misma. Utiliza para ello alguna de las webs indicadas por el profesor para obtener toda la información de las mismas, describiendo los exploits que permiten explotar dicha vulnerabilidad.
    • Explota la vulnerabilidad previamente seleccionada. Usa para ello el framework metasploit (o alguna herramienta similar) presente en la máquina virtual Kali Linux.
    • Busca una solución para una de las vulnerabilidades anteriormente detectada. Llévala a cabo y repite el análisis para verificar que ya no es una vulnerabilidad del equipo.
  • Apartado 2: Vulnerabilidades de aplicaciones web.

Utiliza la máquina virtual Kali Linux en la que puedes encontrar una aplicación web vulnerable llamada dwva, y lleva a cabo las siguientes acciones:

    • Realizar un ataque de Inyección SQL para obtener información sobre el esquema de la base de datos (nombre de la base de datos, nombre de las tablas y las columnas de una de las tablas).
    • Analizar el código PHP del formulario vulnerable a Inyección SQL. ¿Qué cambios harías sobre dicho código para mitigar el ataque? ¿Valdría esta solución adoptada para mitigar una autenticación insegura sobre la aplicación web? Justifica la respuesta.
    • Realizar un ataque Local File Inclusion sobre la aplicación web y describe los resultado obtenidos. Describe como mitigas dicha vulnerabilidad.
  • Apartado 3:Análisis de Metadatos.

Utiliza la herramienta Foca para llevar a cabo un análisis de metadatos. Para ello se pide:

    • Realizar un análisis de metadatos del dominio que desees.
    • Realizar un análisis de metadatos de algunos de tus ficheros locales.
    • Realiza un borrado de los metadatos encontrados en tus ficheros locales usando herramientas como  MAT (Metadata Anonymisation Toolkit).
    • Metadatos en la nube:
      • Describe en qué consiste el servicio de metadatos en la nube conocido como Faast.
      • ¿Qué productos ofrece?
      • Realiza el análisis de algún documento y describe el resultado obtenido.
      • ¿Es posible limpiar los metadatos con alguna de las herramientas ofrecidas por este servicio online? En caso afirmativo, describe el proceso a realizar.
  • Apartado 4: Análisis Forense.
    • Instala y configura una herramienta para la realización de evidencias digitales(por ejemplo, Access Data Forense Toolkit), sobre una máquina Windows del entorno virtual.
    • Crea una imagen como evidencia de una unidad usb sobre la que previamente has debido borrar algunos ficheros que deberás recuperar usando la imagen.
    • Abre una sesión en cualquier sitio web al que debas logarte. Usa la herramienta anterior para capturar toda la información existente en la memoria RAM. Monta dicha captura sobre una imagen y realiza un análisis forense de la imagen creada. Finalmente, captura la pantalla en la que se muestra la sesión abierta en el sitio web junto con las credenciales de acceso.

NOTA IMPORTANTE

Para todos los apartados es necesario entregar las capturas de pantalla de los principales pasos realizados, explicando el proceso seguido en cada uno de ellos. Las capturas de pantalla realizadas deben tener como fondo de pantalla la plataforma con tu usuario mostrando claramente la foto de tu perfil. Aquellos apartados/subapartados que no cumplan esta condición no serán corregidos.
